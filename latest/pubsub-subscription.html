class google.cloud.pubsub.subscription.Subscription(name, topic=None, ack_deadline=None, push_endpoint=None, retain_acked_messages=None, message_retention_duration=None, client=None)[source]#

Bases: object

Subscriptions receive messages published to their topics.

See: https://cloud.google.com/pubsub/docs/reference/rest/v1/projects.subscriptions

Parameters:
  • name (str) – the name of the subscription.
  • topic (google.cloud.pubsub.topic.Topic) – (Optional) the topic to which the subscription belongs; if None, the subscription’s topic has been deleted.
  • ack_deadline (int) – the deadline (in seconds) by which messages pulled from the back-end must be acknowledged.
  • push_endpoint (str) – (Optional) URL to which messages will be pushed by the back-end. If not set, the application must pull messages.
  • retain_acked_messages (bool) – (Optional) Whether to retain acked messages. If set, acked messages are retained in the subscription’s backlog for a duration indicated by message_retention_duration.
  • message_retention_duration (datetime.timedelta) – (Optional) Whether to retain acked messages. If set, acked messages are retained in the subscription’s backlog for a duration indicated by message_retention_duration. If unset, defaults to 7 days.
  • client (Client) – (Optional) The client to use. If not passed, falls back to the client stored on the topic.
